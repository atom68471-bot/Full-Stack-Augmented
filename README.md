# Financial Planner

A simple personal finance tracker with AIŌĆæpowered monthly analysis. Users record incomes and expenses; the backend aggregates data and returns human-readable advice generated by an AI stub or real LLM. This repository contains a Spring Boot REST API (Java) and an Angular 17 frontend.

## Features
- CRUD transactions (date, amount, category, type, description)
- Filter by date, category
- Monthly summary with AI-generated advice
- In-memory persistence (no database)
- Unit tests for backend and frontend
- Dev container configured

## Architecture
### Backend
- Spring Boot application under `backend/`
- Model: `Transaction`
- Repository: in-memory implementation
- Service layer: `TransactionService`, `AnalysisService`
- Controllers: `/api/transactions`, `/api/analysis`
- AI abstraction: `AiClient` interface; initial stub returns hardcoded advice
- Asynchronous analysis with `@Async` returning `CompletableFuture`

### Frontend
- Angular application under `frontend/`
- Components: transaction list, form, analysis view
- Services using `HttpClient` to call backend API
- Models mirror backend DTOs
- Uses `@defer` blocks to lazily render the transaction list and analysis results
- Chart.js (or similar) for spending vs income visualization

### AI Feature
The `/analysis` endpoint accepts year and month, aggregates the user's transactions, and forwards a summary to an `AiClient`. The response contains advice such as "You spent 20% more on dining this month; consider cooking at home." Initially the AI client is a stub; it's easily replaceable with a real LLM by implementing the interface.
The frontend displays the advice alongside charts and summary cards.

## Development
- Dev container defined in `devcontainer.json` for Codespaces or VS Code Remote. Open the repo in GitHub Codespaces and the container will build automatically, installing Java 17, Node 20, Maven, and Angular CLI. The `postCreateCommand` compiles the backend and installs frontend dependencies.
- Use `./mvnw spring-boot:run` (wrapper is now included) or `mvn` and `npm start` during development.
- API base path: `http://localhost:8080/api`
- Frontend served at `http://localhost:4200`

## Continuous Integration
A GitHub Actions workflow (`.github/workflows/ci.yml`) runs automatically on push and pull requests to `main`:
1. Builds and tests the backend using Maven/JDK 17.
2. Installs dependencies and runs Angular unit tests with Node 20.

Code coverage results are printed to the log, and failing tests will block merges.
