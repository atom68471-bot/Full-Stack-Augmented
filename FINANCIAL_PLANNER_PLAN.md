## Plan: Gestor de Finanzas Personales con Consejos

TL;DR: Build a full-stack finance tracker where users record incomes and expenses by category, and an AI feature analyzes trends monthly to offer saving advice. The backend is Spring Boot with in-memory persistence, the frontend is Angular with deferred views and charts. We'll phase the work into setup, core CRUD backend, AI analysis, frontend scaffolding/UI, testing/TDD, documentation, and environment setup. A markdown version of this plan will live in the repo for reference.

**Steps**
1. **Project scaffolding**
   * Create Spring Boot starter project with Web and Lombok dependencies plus test starters.
   * Add Angular workspace using CLI (`ng new`) with minimal routing; set up a service to call backend.
   * Add `devcontainer.json` for Java/Node Angular environment.
   * Configure Maven/Gradle build to allow in-memory repository (a simple `List` or `Map`).
2. **Backend core entities and controllers**
   * Define `Transaction` model (id, date, amount, category, type INCOME/EXPENSE, description).
   * Create repository interface and in-memory implementation.
   * Implement service layer (`TransactionService`) with CRUD methods.
   * Build REST controller `/transactions` exposing endpoints to list, add, update, delete, filter by date/category.
   * Add basic validation.
3. **AI feature: monthly analysis**
   * Add endpoint `/analysis` that accepts year and month; aggregates transactions and sends summary to an LLM.
   * Define `AnalysisRequest` and `AnalysisResponse` DTOs; mock the LLM call with a simple stub initially (e.g., hardcoded textual advice).
   * Later swap the stub for a real call to an OpenAI-like API; design interface `AiClient` to abstract over provider.
   * Ensure analysis runs asynchronously (e.g., `@Async` or CompletableFuture).
4. **Frontend scaffolding and UI**
   * Create Angular models mirroring backend DTOs.
   * Implement a service `TransactionService` using `HttpClient` to call the API.
   * Build main components: `transaction-list`, `transaction-form`, `analysis-view`.
   * Use `@defer` blocks where long lists or charts appear (list of transactions, analysis results).
   * Add Chart.js or similar to display monthly spending vs income.
   * Include filters by date range and category.
5. **Testing strategy**
   * Backend: write unit tests for service and controller using Mockito and Spring Boot test; start with tests generated by Copilot via comment prompts.
   * AI endpoint tests: mock `AiClient` to verify behavior and advice content handling.
   * Frontend: use Angular testing with Jasmine/Karma; write component tests and service tests, following TDD with Copilot suggestions.
   * Achieve ~100% coverage for core modules.
6. **Documentation & run instructions**
   * Add README section describing app purpose, architecture, and AI feature.
   * Create `RUNNING.md` or similar with steps to start backend (`./mvnw spring-boot:run`), frontend (`npm start`), and example data insertion (cURL commands or JSON fixtures).
   * Provide sample transactions for testing and explain how to invoke analysis.
7. **Dev environment & reproducibility**
   * Finalize `devcontainer.json` with Java 17, Node 18/20, Angular CLI; include Maven/Gradle and npm installation instructions.
   * Ensure GitHub Codespaces opens with the project ready to build and run.
   * Add GitHub Actions workflow for building and running backend tests and frontend tests on push.
8. **Polish & review**
   * Refactor per Copilot Chat suggestions, remove boilerplate, keep code clean.
   * Add comments, Javadoc, and inline documentation.
   * Update plan file in repo.

**Verification**
1. Build backend and call CRUD endpoints with Postman/cURL, confirm in-memory persistence.
2. Trigger analysis endpoint with sample data; check AI advice format.
3. Run Angular app, add transactions, view list, filter, and see analysis and charts.
4. Run all unit tests (backend & frontend) and confirm coverage reports.
5. Open project in Codespaces and verify both services start without additional setup.
6. Peer review using rubric; ensure each trait is addressed in code and docs.

**Decisions**
- Use in-memory storage (no DB) per instruction to simplify setup.
- AI client abstracted early to allow real API swap.
- Work phases are independent; backend core completed before frontend ensures clear API contract.

**Further Considerations**
1. Should we support user accounts or keep it single-user? (Scope down: single-user to reduce complexity.)
2. How complex should the AI advice be? Start simple (e.g., "you spent X% more on dining"), then refine.
3. Frontend charting library choice (Chart.js vs. ngx-charts) — Chart.js is lightweight.
